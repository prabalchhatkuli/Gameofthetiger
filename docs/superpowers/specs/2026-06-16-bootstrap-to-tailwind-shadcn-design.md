# Bootstrap → Tailwind + shadcn/ui Migration — Design

**Date:** 2026-06-16
**Status:** Approved (pending spec review)

## Goal

Replace the frontend's Bootstrap / React-Bootstrap UI layer with **Tailwind CSS v4 + shadcn/ui**, in a single big-bang migration that removes Bootstrap entirely. Components adopt shadcn's default look. Game logic, the AI engine, routing, and Firebase are untouched.

## Decisions (from brainstorming)

1. **Big-bang** — set up the new stack, migrate all Bootstrap-using components, and remove Bootstrap in one change. Avoids the Tailwind-preflight vs Bootstrap-reset conflict that an incremental approach would create.
2. **Adopt shadcn's default look** — no separate redesign; components take shadcn's neutral modern styling out of the box.
3. **Tailwind v4** via the `@tailwindcss/vite` plugin.
4. **shadcn in JSX mode** (`components.json` `"tsx": false`) so generated components are `.jsx`, matching the codebase. No TypeScript conversion.
5. **`game.css` and the board/square components stay as-is** — custom positioning CSS, not Bootstrap; the board is where pixel-exact layout matters.

## Non-Goals

- No dark mode / theme switching (light only).
- No restyle of the game board or `game.css`.
- No changes to game logic, AI (`src/ai/`), routing, sockets, or Firebase.
- No responsive hamburger menu for the navbar (a simple responsive flex nav is sufficient; a mobile drawer is a future nice-to-have).
- No new UI test framework (verification is build + manual pass; the existing Vitest AI suite must stay green).

## Tooling setup

### Tailwind v4 + Vite
- Install `tailwindcss` and `@tailwindcss/vite`.
- `frontend/vite.config.js`: add the `tailwindcss()` plugin (from `@tailwindcss/vite`) to `plugins`, and add a path alias:
  ```js
  resolve: { alias: { '@': '/src' } }   // or path.resolve(__dirname, './src')
  ```
- Global stylesheet (e.g. `src/index.css`) begins with `@import "tailwindcss";` plus the shadcn theme variables (generated by shadcn init). `src/index.jsx` imports this file **instead of** the Bootstrap CSS.

### Path alias for shadcn imports
shadcn components import from `@/components/ui/*` and `@/lib/utils`. Add to `frontend/tsconfig.json`:
```json
"compilerOptions": {
  "baseUrl": ".",
  "paths": { "@/*": ["./src/*"] },
  ...existing options
}
```
(Keep the existing tsconfig options; just add `baseUrl` + `paths`. The Vite alias above makes it work at build/runtime; the tsconfig entry makes editor/CLI resolution work. No separate `jsconfig.json` to avoid dual-config confusion.)

### shadcn init (JSX)
- Run `npx shadcn@latest init`, choosing the JavaScript (not TypeScript) option so `components.json` has `"tsx": false`; base color neutral; CSS file `src/index.css`; alias `@/components` and `@/lib/utils`.
- This generates `src/lib/utils.js` (the `cn` helper = `clsx` + `tailwind-merge`), `components.json`, and writes shadcn's CSS variables into the global stylesheet. It adds deps `class-variance-authority`, `clsx`, `tailwind-merge`, and the v4 animation helper (`tw-animate-css`).
- Generate the needed primitives: `npx shadcn@latest add button dialog tabs input label radio-group` → creates `src/components/ui/{button,dialog,tabs,input,label,radio-group}.jsx` and pulls their Radix deps.

## Component mapping

| Bootstrap | shadcn / Tailwind replacement |
|---|---|
| `react-bootstrap/Button` | `@/components/ui/button` `Button` (variant props for primary/secondary/etc.) |
| `react-bootstrap/Modal` | `@/components/ui/dialog` `Dialog` + `DialogContent`/`DialogHeader`/`DialogTitle`/`DialogFooter` |
| `react-bootstrap/Tabs`+`Tab` | `@/components/ui/tabs` `Tabs`/`TabsList`/`TabsTrigger`/`TabsContent` |
| `react-bootstrap/Form` + `form-control` | `@/components/ui/input` `Input` + `@/components/ui/label` `Label` |
| `custom-control` radio group | `@/components/ui/radio-group` `RadioGroup`/`RadioGroupItem` + `Label` |
| `react-bootstrap/Nav`+`Navbar` | Tailwind `<nav>` (flex row, responsive wrap) + shadcn `Button`/links |
| `react-bootstrap/Image` | `<img>` with Tailwind classes |
| `container` | `mx-auto max-w-… px-…` |
| `text-center` | `text-center` |
| `text-success` / `text-warning` / `text-light` | `text-green-600` / `text-yellow-500` / `text-white` (tune to shadcn palette) |
| `d-inline-block` | `inline-block` |

## Per-file changes

- **`navbar.component.jsx`** — replace `Navbar`/`Nav` with a Tailwind `<nav>` (flex, items spaced, wraps on small screens); links become styled anchors / shadcn `Button` (Play the Game, Instruction, About, Profile, Log In / Signout). Keep the goat/tiger SVG brand images and the `signOut(auth)` handler. (No collapsible hamburger — simple responsive flex.)
- **`landingpage.component.jsx`** — `Image` → `<img className="…">`; hero text/wrapper divs use Tailwind. Welcome banner keeps its meaning (`text-light bg-success` → Tailwind equivalents).
- **`loginpage.component.jsx`** — `Form`/`Form.Group`/`Form.Control` → `Label` + `Input`; submit `Button` → shadcn `Button`. Keep `handleLogin` (uses `signInWithEmailAndPassword`) and `enterPressed`.
- **`signup.component.jsx`** — same Form → `Input`/`Label`/`Button`; keep `handleSignup` (uses `createUserWithEmailAndPassword` + `generateUserDocument`).
- **`chat/multichoice.component.jsx`** — `Modal` → `Dialog`; `Tabs`/`Tab` → shadcn `Tabs`; the goat/tiger `custom-control` radios → `RadioGroup`/`RadioGroupItem`; `Button`s → shadcn. **Cleanup:** the current `ReactDOM.render(...)` calls that inject the generate-link result/error into a `<div id="generate result">` are replaced with React state (the component is being rewritten anyway; this removes a deprecated imperative pattern). Keep the `onGenerateButtonClick`/`createGame`/`joinGame` logic and the auth-token-bearing axios calls.
- **`chat/gameroom.component.jsx`** — `Button` → shadcn `Button`; keep `validateRoom`/`startGame` logic and token handling.
- **`game/winner.component.jsx`** — `Modal` → `Dialog` configured non-dismissible (no close on overlay/esc; matches the current `backdrop="static" keyboard={false}`); keep the "Understood" redirect button. (Note: the unused `Tabs`/`Tab` imports here are dropped.)
- **`profile.component.jsx`** — the AI-stats `<table>` (added in the AI feature) gets Tailwind classes for borders/spacing to match the new look. Player-info section uses Tailwind text utilities.
- **`index.jsx`** — remove `import 'bootstrap/dist/css/bootstrap.min.css'`; add `import './index.css'` (the Tailwind/shadcn stylesheet).
- **Audit pass** — grep the remaining components (`about`, `instruction`, `chat/chat`, `gamechoice`) for stray Bootstrap utility classes (`container`, `btn`, `text-*`, etc.) and convert any found to Tailwind.
